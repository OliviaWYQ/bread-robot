# LeRobot平台能力对齐矩阵

本文档明确哪些功能LeRobot原生支持，哪些需要定制改造。

## 一、LeRobot平台概述

LeRobot是由Hugging Face推出的开源机器人学习平台，提供：
- 多种开源机械臂硬件参考设计（如so100）
- 数据采集和示教工具
- 预训练模型和策略训练框架
- 运动控制基础设施
- 视觉识别集成

## 二、能力对齐矩阵

### 运动控制能力

| 功能 | LeRobot支持 | 需要定制 | 说明 |
|-----|------------|---------|-----|
| 6自由度机械臂运动规划 | ✅ 原生支持 | 无 | 平台原生支持逆运动学和路径规划 |
| 示教复刻工作流 | ✅ 原生支持 | 无 | LeRobot核心特性，支持一键采集示教数据 |
| 夹爪开合控制 | ✅ 原生支持 | 无 | 标准夹爪完全支持 |
| 力反馈与力控 | ✅ 框架支持 | 需要硬件适配 | 平台框架支持力控，需要传感器硬件 |
| 末端执行器更换 | ❌ 不支持 | 需要定制开发 | 需要设计自动换头机构和机械结构 |
| 高速运动 | ✅ 支持 | 无 | 基于所用电机性能 |
| 碰撞检测 | ✅ 框架支持 | 需要适配 | 可通过电流检测或额外传感器实现 |

### 视觉感知能力

| 功能 | LeRobot支持 | 需要定制 | 说明 |
|-----|------------|---------|-----|
| 物体检测与定位 | ✅ 原生支持 | 少量标注数据 | 集成了SOTA视觉模型，可以微调 |
| 位姿估计 | ✅ 支持 | 需要标定 | 平台支持相机标定和位姿计算 |
| 面团发酵程度检测 | ❌ 不支持 | 需要数据训练 | 需要收集不同发酵程度数据训练自定义模型 |
| 多相机标定 | ✅ 支持 | 需要标定工作 | 平台支持多相机配置 |

### 任务编排能力

| 功能 | LeRobot支持 | 需要定制 | 说明 |
|-----|------------|---------|-----|
| 序列动作执行 | ✅ 原生支持 | 无 | 可以执行示教好的动作序列 |
| 条件分支判断 | ✅ 支持 | 需要业务逻辑开发 | 框架支持基于传感器反馈的分支 |
| 状态机管理 | ✅ 支持 | 需要自行实现 | 需要基于LeRobot API开发状态机 |
| 长时间等待计时 | ✅ 支持 | 无 | 可以实现定时等待 |
| 异常中断恢复 | ✅ 支持 | 需要业务逻辑开发 | 需要自行编写恢复逻辑 |

### 领域特定功能

| 功能 | LeRobot支持 | 需要定制 | 说明 |
|-----|------------|---------|-----|
| 原料自动称量 | ❌ 完全不支持 | 重度定制 | 需要集成电子秤，定制称量机构 |
| 揉面动作执行 | ❌ 需要定制末端 | 需要定制揉面勾末端 | 现有夹爪无法揉面，必须更换末端 |
| 烤箱门操作 | ✅ 运动能力支持 | 需要力控适配 | 运动能力足够，需要力控参数调节 |
| 温度读取监控 | ❌ 不支持 | 需要扩展IO | 需要添加温度传感器驱动 |
| 食品安全设计 | ❌ 不相关 | 需要机械设计 | 所有接触食品部件需要食品级材料设计 |
| 高温防护 | ❌ 不相关 | 需要机械设计 | 机械臂末端需要隔热处理 |

## 三、定制开发工作量估算

| 模块 | 工作量（人天） | 难度 | 关键技术点 |
|-----|--------------|------|-----------|
| 自动换末端机构设计加工 | 3-5 | 中等 | 机械设计，快速连接 |
| 定制揉面勾末端设计加工 | 1-2 | 低 | 食品级材料，形状设计 |
| 烤盘夹爪隔热改造 | 1 | 低 | 隔热材料处理 |
| 温度传感器IO扩展 | 2-3 | 中等 | 硬件接线，驱动开发 |
| 换头流程软件开发 | 2 | 中等 | 状态机集成 |
| 揉面动作示教调试 | 3-5 | 中等 | 力度参数调试 |
| 烤箱门开关力控调试 | 2 | 中等 | 力控参数适配 |
| 全流程联调整合 | 5-7 | 高 | 各个模块衔接调试 |

**总计定制工作量：** 17-25人天

## 四、结论

1. **基础运动控制部分**：LeRobot平台已经提供了完整的支持，不需要重复造轮子
2. **核心挑战在于定制末端和食品领域特定设计**：需要做一定的机械改造和软硬件扩展
3. **MVP可以通过用户预处理简化复杂度**：由用户完成称量，聚焦验证自动化流程可行性
4. **技术风险可控**：大部分定制工作是机械设计和集成，算法层面LeRobot已经提供了足够支持
