# 做面包场景算法改造评估

## 做面包流程分析

典型做面包主要步骤：
1. 配料称量（面粉、水、酵母、盐、糖、黄油）
2. 混合揉面
3. 一次发酵
4. 排气整形
5. 二次发酵
6. 烘烤

其中需要机器人操作的关键难点：**柔性物体（面团）操作**和**高精度称量**，发酵和烘烤是和家电交互。

---

## 1. 柔性物体（面团）操作算法适配

### 问题分析

面团是典型的**粘弹性柔性物体**，特性：
- 形状会随操作变化，不可刚体假设
- 操作中接触力变化大，需要力反馈
- 不同面团湿度、温度不同，物理参数变化大
- 揉面、整形动作是持续接触交互，不是单次抓取放置

### LeRobot 现状

LeRobot 公开数据集中，柔性物体操作案例有限，现有算法（ACT）主要针对刚体操作设计。

### 改造方向

#### （1）感知层面改进

- **增加力感知**: 末端需要六维力传感器，LeRobot 本身支持读取力矩数据作为观测输入
- **高频率视觉**: 面团形变快，需要更高帧率（30fps+）视觉反馈
- **稠密预测**: 原来ACT只预测关节角，现在需要预测接触区域形变，可考虑添加语义分割预测面团区域

#### （2）算法层面改进

| 方案 | 思路 | 复杂度 |
|------|------|--------|
| **增加示教数据多样性** | 收集不同硬度、湿度面团的示教数据，让模型泛化 | ⭐⭐ 低 |
| **力-视觉多模态输入** | 将关节力矩/末端力数据和视觉拼接作为ACT编码器输入，让模型学习根据力调整动作 | ⭐⭐⭐ 中 |
| **迭代闭环控制** | 缩小动作chunk长度，每步都重新观测，根据当前面团形状调整动作，减少误差积累 | ⭐⭐ 低 |
| **结合动力学模型** | 用NeRF或其他方法建模面团形变，在动作预测中考虑形变预测 | ⭐⭐⭐⭐⭐ 很高 |
| **Diffusion Policy 替换ACT** | 研究显示DP对形变更鲁棒，可以直接尝试DP训练对比 | ⭐⭐ 中 |

**推荐路线**: 先做多模态输入（视觉+力）+ 增加数据多样性，如果效果不够再换Diffusion Policy。

#### （3）硬件设计配合

- 揉面需要专用末端执行器（刮刀/钩型揉面器），不是标准夹爪
- 揉面容器需要固定定位

### 评估结论

- 技术风险：中高，柔性物体本身是机器人学开放问题
- 可落地性：MVP可以通过足够多的示教数据+力反馈达到可用效果
- 改造工作量：算法改动不大，主要是数据采集和硬件适配

## 2. 高精度称量动作精度优化

### 问题分析

称量要求精度通常：
- 面粉：±5g （总重500g，误差1%）
- 酵母/盐：±1g （总重5-10g，误差10%）
LeRobot 默认模仿学习开环执行，误差通常在毫米级到厘米级，不满足称量精度要求。

### 误差来源分析

1. **机械本身重复定位误差**: 即使高精度臂也有 ±0.1~0.3mm
2. **模仿学习预测误差**: 策略网络输出的关节位置误差
3. **面粉/干料流动不确定性**: 倾倒量难以精确控制
4. **容器定位误差**: 每次放容器位置不一致

### 改进方案

#### 方案一：闭环反馈称量（推荐MVP采用）

架构：称重传感器 → 误差反馈 → 增量倾倒动作调整

```
步骤：
1. 整体容器放在高精度电子秤上
2. 机器人开始倾倒粉料/液体
3. 实时读取电子秤读数
4. 当接近目标重量时，减慢倾倒速度
5. 达到目标重量立即停止倾倒
```

适配LeRobot:
- 电子秤通过串口/蓝牙和主控电脑通信，重量作为观测输入给策略网络
- 可以分解为多个增量倾倒动作，每次倒一点然后闭环调整
- 训练: 示教不同初始量下的倾倒过程，让模型学习根据当前重量差调整动作

复杂度: ⭐⭐ 中等，需要额外传感器，但算法改动小。

#### 方案二：开环精确建模

对每种物料的倾倒流量建模，根据目标重量计算倾倒时间和角度。

问题: 物料粘度、湿度变化影响流量，开环很难稳定精度。不推荐MVP使用。

#### 方案三：视觉辅助估算

用相机看容器内物料高度估算重量，配合调整。精度不足以满足小重量（盐/酵母）要求，可作为辅助。

### 精度优化总结

| 优化项 | 措施 |
|--------|------|
| 硬件选择 | 选择重复定位精度更高的机械臂（UR5e/Franka，优于Koch） |
| 传感器增补 | 增加一个高精度蓝牙电子秤（0.1g精度）作为闭环反馈 |
| 算法 | 分层策略：粗倒+精调，增量式动作，实时重量闭环 |
| 标定 | 对容器位置进行视觉自动标定，减少定位误差 |

### 评估结论

- 技术风险：低，闭环称量方案是工业常用方法，可行性高
- 改造工作量：主要是硬件集成和数据采集，算法改动小
- 预期精度：可以达到 ±1g 满足要求

## 3. 发酵/烘烤环节和厨房家电交互方案

### 问题分析

发酵需要控制温度湿度（通常28℃，75%RH，1-2小时），烘烤需要控制温度时间（180-200℃，20-30分钟），这些功能厨房已有成熟家电（烤箱、发酵箱），机器人不需要自己做温度控制，只需要解决**交互（存取面团）**和**状态同步**。

### 交互场景

1. 一次发酵完成 → 机器人从发酵箱取出面团 → 整形 → 放入烤盘 → 送入二次发酵
2. 二次发酵完成 → 机器人送入烤箱 → 烘烤完成 → 机器人取出面包

### 交互方案选型

#### 方案A：机械开门方案（完全自动化，推荐MVP）

- 思路: 机器人自己操作家电门把手开门，取出/放入，关门
- 实现:
  - 对烤箱/发酵箱门位置进行标定，示教开门轨迹
  - 大多数烤箱门把手是旋转或横拉，机器人可以完成
  - 门需要能够保持打开状态（或者用简易机械保持）
-  Pros: 完全自动化，不需要改造家电
-  Cons: 需要额外示教开门动作，增加机械行程，门如果有弹力需要力控制

#### 方案B：自动门改造家电

- 思路: 改装发酵箱/烤箱门，添加电动开关门
-  Pros: 机器人操作简单，只需要取放
-  Cons: 需要改造家电，增加成本和复杂度，MVP不推荐

#### 方案C：人机协作（简化MVP可选）

- 思路: 发酵/烘烤等待过程中，通知人来完成存取，机器人只做前面工序
-  Pros: 机器人不需要操作门，减少复杂度
-  Cons: 不是全自动化，用户体验差

### 状态监控方案

- **温度**: 可以依赖家电自身控温，机器人只需要计时和触发下一步
- **是否完成**: 基于定时即可（发酵时间和烘烤时间是配方固定的）
- 如果需要监控，可以加一个温度探针放在发酵箱/烤箱内，有线传输数据

### 通信接口

- 大多数家用智能烤箱/发酵箱有WiFi，可以通过MQTT或HTTP API控制
- 如果是非智能家电，不需要通信，机器人只需要计时，机械存取即可
- 推荐MVP采用"机械操作+定时"方案，不需要智能家电改造

### 交互方案结论

| 方案 | 自动化程度 | 复杂度 | 推荐MVP |
|------|------------|--------|---------|
| 机器人开门存取 | 100% | ⭐⭐ 中 | ✅ 是 |
| 电动门改造 | 100% | ⭐⭐⭐⭐ 高 | ❌ 否 |
| 人机协作 | 半自动 | ⭐ 低 | 备选 |

技术风险低，主要是示教轨迹要准确，开门动作需要力控配合（如果用Franka很容易实现）。
