# 技术选型方案

## 1. 硬件选型与传感器增补

### 基础机械臂选型

| 选项 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Franka Emika Panda** | 原生力矩传感器，支持拖动示教，力控精度高 | 价格高（~10万RMB） | ⭐⭐⭐⭐ 科研首选 |
| **Universal Robots UR5e** | 精度高（±0.03mm），成熟稳定，支持力控 | 价格较高（~8万RMB） | ⭐⭐⭐⭐ 工业首选 |
| **LeRobot Koch v1** | 开源，成本低（~2万RMB） | 无原生力传感器，需要额外加 | ⭐⭐⭐ 低成本方案 |

**MVP推荐**: 如果预算允许优先Franka Panda，力控对揉面和开门都很有帮助。Koch可以作为低成本方案，但需要额外加力传感器。

### 末端执行器

- **基本夹爪**: Robotiq 2F-85 或 RG2，可靠，支持力控
- **揉面末端**: 需要定制 3D 打印钩型揉面刀，安装在夹爪上
- **倾倒工具**: 定制勺/杯状工具，配合夹爪夹持配料容器
- **总数量**: 1个主夹爪 + 可换末端，或者两个夹爪（不推荐，增加复杂度）

### 必须新增传感器

| 传感器 | 作用 | 规格推荐 | 必要性 |
|--------|------|----------|--------|
| **高精度电子秤** | 称量闭环反馈 | 精度 0.1g，最大量程 5kg，蓝牙/串口通信 | ✅ 必须 |
| **六维力传感器** | 面团揉面力反馈，开门力控制 | ATI Mini45 或 Robotiq FT 300，如果机械臂不带 | ✅ 必须（如果机械臂原生不带） |
| **额外RGB-D相机** | 除了全局和手腕，容器定位和面团状态观测 | Intel RealSense D435i | ✅ 推荐，三个相机足够 |
| **温度传感器** | 发酵箱/烤箱温度验证 | 单通道热电偶，max 300℃ | ⭕ 可选，MVP可以省略 |

### 算力配置

- **训练**: 本地工作站 NVIDIA RTX 4090 (24GB) 足够，ACT训练单卡可以
- **推理**: 本地工作站推理足够，不需要边缘端，因为是固定工作站
- **主控电脑**: Ubuntu 22.04，32GB+ 内存

## 2. 模型选型与微调方案

### 基础模型选择

LeRobot 提供两种主流模仿学习算法：

| 算法 | 适用场景 | 推荐理由 |
|------|----------|----------|
| **ACT (Action Chunking with Transformers)** | 默认首选，训练稳定，社区使用多 | MVP baseline，先从ACT开始 |
| **Diffusion Policy** | 高精度轨迹，对动态场景更鲁棒 | 如果ACT对柔性物体效果不好，再切DP |

**MVP路线**: 从ACT开始，快速跑通 baseline，效果不好再换Diffusion Policy，这样风险低。

### 微调策略

| 层次 | 方案 | 计算量 |
|------|------|--------|
| **全参数微调** | 从ACT预训练权重（LeRobot官方）开始，全参数微调 | 中等，RTX 4090 几个小时到一天 |
| **LoRA微调** | 只微调注意力层低秩矩阵，冻结主干 | 更快，显存占用更低，适合数据量不大时 |
| **特征提取层冻结** | 冻结ViT视觉编码器，只训练Transformer解码器 | 最快，适合数据很少时 |

**MVP推荐**: 从全参数微调开始，因为我们会采集足够的任务特异性数据，全微调效果最好。

### 预训练迁移

LeRobot 官方已经有大型机器人数据集预训练的ACT权重，可以直接用来初始化，比从零开始训练收敛更快，泛化更好。所以不需要从零开始训练。

## 3. 数据采集方案

### 需要采集的任务分解

我们需要把面包制作分解为多个子任务，分别采集示教数据：

| 子任务 | 示教次数估计 | 说明 |
|--------|--------------|------|
| 1. 从存储盒取出配料容器 | 20次 | 不同初始位置 |
| 2. 称量倾倒：面粉 | 30次 | 不同目标重量，不同初始重量 |
| 3. 称量倾倒：酵母/盐/糖 | 20次每次 | 小重量精度要求高 |
| 4. 称量倾倒：水/液体 | 20次 | 液体流动性不同 |
| 5. 混合揉面 | 50次 | 不同硬度面团，不同揉法 |
| 6. 面团排气整形 | 30次 | 不同大小面团 |
| 7. 放入烤盘/发酵篮 | 20次 | 定位要求 |
| 8. 发酵箱开门/放进去/关门 | 20次 | 力控开门 |
| 9. 发酵箱取出 → 烤箱放入 | 20次 | 转移 |
| 10. 烤箱开门/取出面包 | 20次 | 高温注意安全 |

**总预计示教次数**: ~250-300次，总时间大概 2-3 天采集。

### 采集流程建议

1. **环境固定**: 工作站所有设备位置固定，容器统一规格，减少方差
2. **分任务采集**: 一个子任务一个数据集，分开训练，最后可以做成流水线
3. **数据多样化**:
   - 面团不同湿度硬度
   - 容器不同初始位置
   - 不同目标重量
   这样模型泛化更好
4. **质量检查**: 采集完每条轨迹都可视化检查，删除失败的轨迹
5. **版本管理**: 用 Hugging Face Hub 做版本管理，方便实验对比

### 数据标注

模仿学习不需要额外标注，拖动示教记录的数据直接可以训练。只需要对失败轨迹做过滤即可。

### 数据增强

- 关节空间高斯噪声注入
- 视觉随机遮挡增强
- 不同光照条件采集

## 4. 软件架构方案

整体软件架构沿用 LeRobot 原生架构，只做增量扩展：

```
┌─────────────────────────────────────────────────────────┐
│  任务调度层 (Python)                                      │
│  - 按配方流程顺序调度各个子任务                          │
│  - 读取电子秤重量，处理状态机                            │
├─────────────────────────────────────────────────────────┤
│  LeRobot 策略层                                         │
│  - ACT/DP 模型                                          │
│  - 多模态输入: 视觉 + 关节 + 力 + 重量                  │
├─────────────────────────────────────────────────────────┤
│  硬件驱动层                                             │
│  - 机械臂驱动 (原生LeRobot)                              │
│  - 相机驱动 (原生LeRobot)                                │
│  - 电子秤驱动 (新增: 串口读取)                           │
│  - 力传感器驱动 (如果外接: 新增)                         │
└─────────────────────────────────────────────────────────┘
```

### 第三方依赖

- 保持 LeRobot 官方依赖不变
- 新增: pyserial (电子秤串口通信)
- 新增: bluepy (如果电子秤是蓝牙)

## 5. 选型总结

| 项 | 选型结论 |
|----|----------|
| 机械臂 | 优先 Franka Panda / UR5e，低成本选 Koch |
| 传感器 | 必须加 0.1g 精度电子秤 + 六维力（如无原生） |
| 算法 | baseline 用 ACT，效果不好换 Diffusion Policy |
| 模型 | 从LeRobot官方预训练全参数微调 |
| 数据 | 分任务采集，总共 ~250-300 次示教 |
