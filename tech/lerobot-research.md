# LeRobot平台能力调研

## 1. 平台概述

LeRobot 是由 Hugging Face 推出的开源机器人学习平台，旨在降低模仿学习在机器人领域的应用门槛，提供标准化的硬件设计、软件栈和预训练模型，支持快速构建机器人应用。

## 2. 硬件配置支持

LeRobot 官方支持多种机械臂配置：

### 官方支持硬件

| 硬件 | 规格 | 支持状态 |
|------|------|----------|
| **Koch v1 机械臂** | 7自由度，基于uFactory xArm定制改造，最大负载 5kg，重复定位精度 ±0.1mm | ✅ 官方首选支持 |
| **UR5e** | Universal Robots 协作机器人，6自由度，负载 5kg，精度 ±0.03mm | ✅ 完全支持 |
| **Franka Emika Panda** | 7自由度，负载 3kg，精度 ±0.1mm，内置力矩传感器 | ✅ 完全支持 |
| **Sawyer** | Rethink Robotics，7自由度，负载 4kg | ✅ 支持 |
| **Kinova Gen3** | 7自由度，负载 2kg | ✅ 支持 |

### 标准化组件

- **末端执行器**: 官方提供基于RG2的夹爪适配，支持电动夹爪和吸盘
- **相机配置**: 推荐 2-3 台 Intel RealSense D435i/D455 （立体深度+RGB），一个全局视角，一个手腕视角
- **算力配置**: 推荐 NVIDIA RTX 3090/4090 或 A10G 进行训练，推理可在本地Jetson Orin运行
- **关节驱动**: 支持绝对编码器，支持关节力矩反馈（Frankana/Panda原生支持）

## 3. 软件栈架构

LeRobot 采用模块化设计，基于 Python 生态构建：

```
┌─────────────────────────────────────────────────────────┐
│   Hugging Face Hub  ← 数据集和预训练模型版本管理       │
├─────────────────────────────────────────────────────────┤
│   LeRobot 库  ← 数据处理、训练循环、评估               │
│   - torch 深度学习框架                                  │
│   - huggingface datasets 数据加载                      │
│   - huggingface hub 模型托管                           │
├─────────────────────────────────────────────────────────┤
│   机器人驱动层 ← 各厂商硬件抽象适配                     │
│   - foxy: for Franka Emika Panda                       │
│   - ur: for Universal Robots                           │
│   - koch: 自定义机械臂驱动                             │
├─────────────────────────────────────────────────────────┤
│   控制层 ← 运动控制、轨迹跟踪                           │
│   - 基于关节空间控制                                    │
│   - 末端笛卡尔空间轨迹跟踪                              │
└─────────────────────────────────────────────────────────┘
```

### 核心功能

- **数据格式标准化**: `lerobot_dataset` 格式统一存储示教数据
- **代码版数据卡片**: 每种数据集包含数据加载代码，易于复现
- **ACT 算法原生支持**: Action Chunking with Transformers，当前SOTA模仿学习算法
- **Diffusion Policy 支持**: 另一种主流策略学习算法可选
- **DRM 动态恢复模型**: 支持实时轨迹修正，应对干扰

## 4. 运动控制算法

LeRobot 主要聚焦**模仿学习**（Imitation Learning），而非传统运动规划：

### 默认算法：ACT（Action Chunking with Transformers）

- **核心思想**: 一次性预测未来 N 个动作（chunk），而不是单步预测，减少误差积累
- 典型输出窗口: 8-20 个时间步（每步 50-100ms）
- 架构: ViT 视觉编码器 + Transformer 解码器生成动作序列
- **优势**: 训练稳定，泛化性好，对视觉变化鲁棒
- **局限性**: 对于高精度开环动作，可能存在累积误差

### 支持算法选型

| 算法 | 适用场景 | LeRobot支持 |
|------|----------|-------------|
| ACT | 大多数操作任务，复杂接触，鲁棒性好 | ✅ 首选默认 |
| Diffusion Policy | 高精度路径跟踪，对噪声更鲁棒 | ✅ 支持 |
| IDM (Imitative Dynamical Model) | 动力学建模，用于规划 | ✅ 实验支持 |
| DRM (Dynamic Recovery Model) | 在线修正轨迹，应对干扰 | ✅ 支持 |

### 运动控制特性

- 关节空间控制（位置模式）为主
- 支持阻抗控制（依赖硬件支持，如Franka）
- 开环执行预测轨迹，不做在线碰撞检测（需要额外传感器支持）
- 基于视觉的闭环，通过每步重新观察更新动作预测

## 5. 示教能力

LeRobot 的核心优势是**低成本快速示教**:

### 支持示教方式

1. **拖动示教**（首选）
   - 依赖机械臂支持力控/零力模式（如Franka、UR5e）
   - 人直接抓着末端拖动完成动作，系统记录关节位置 + 视觉 + 夹爪状态
   - 优点: 直观，成本低，适合非专家
   - 缺点: 依赖硬件支持，高精度操作可能人难稳定拖动

2. **键盘遥操作**
   - 通过键盘控制关节增量移动
   - 适合小范围精细调整
   - 缺点: 慢，不适合长轨迹

3. **遥操作设备**
   - 支持 Oculus VR 手柄遥操作映射
   - 支持 haptic device 力反馈遥操作
   - 优点: 远距离示教，人可以保持自然姿势
   - 缺点: 需要额外硬件

### 示教数据管理

- 自动同步所有传感器数据: 关节位置、夹爪位置、所有相机RGB/深度、时间戳对齐
- 一键上传到 Hugging Face Hub 版本管理
- 支持数据滤波和轨迹重采样
- 支持多任务数据混合训练

### 示教效率

- 简单任务（抓取放置）: 10-20 次示教即可收敛
- 复杂操作任务: 50-100 次示教达到可用精度

## 6. 现有能力总结

| 能力维度 | 现状 | 适配面包机器人难度 |
|----------|------|------------------|
| 多自由度机械臂支持 | 多品牌完全支持 | ⭐ 容易 |
| 视觉感知（RGB-D） | 原生支持多相机 | ⭐ 容易 |
| 模仿学习框架 | ACT/Diffusion Policy 开箱即用 | ⭐ 容易 |
| 拖动示教数据采集 | 一键录制自动同步 | ⭐ 容易 |
| 柔性物体操作 | 社区案例少，算法需要适配 | ⭐⭐⭐ 困难 |
| 高精度称量 | 无原生支持，需要传感器和算法扩展 | ⭐⭐ 中等 |
| 外部设备交互 | 原生不支持，需要集成 | ⭐⭐ 中等 |
